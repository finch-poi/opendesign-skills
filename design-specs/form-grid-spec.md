## 一、表单在组件体系中的位置

表单（Form）与 Card、List、Table、Collapse、Dialog 并列，属于**页面容器级**组件。

---

## 二、表单布局模式

### 2.1 三种布局类型

| 布局类型 | `layout` 值 | 适用场景 |
|---------|------------|---------|
| 水平布局 | `h` | 标签在输入框左侧，PC 端主要形式 |
| 垂直布局 | `v` | 标签在输入框上方，移动端主要形式 |
| 行内布局 | `inline` | 紧凑筛选栏、搜索表单 |

### 2.2 常见表单使用场景（原文编号）

根据设计文档示例，表单常见场景包括：

1. **通用信息录入表单** — 多字段组合，标准水平/垂直布局
2. **复杂结构化表单** — 包含分组、嵌套、条件显示的复杂表单
3. **向导式（步骤）表单** — 分步骤填写，每步聚焦少量字段
4. **上传类表单** — 包含文件上传区域（`.docx`、`.xls`、`.ppt` 等格式，≤1M）

### 2.3 表单内支持的控件类型

`OInput`、`OInputNumber`、`OTextarea`、`OSelect`、`OCheckboxGroup`、`ORadioGroup`、`OUpload`

## 三、PC 与 PhonePad 的间距规范

### 3.1 水平标签模式（PC，>840px）

```
┌─────────────────────────────────────────────────┐
│ [标签文字:] [____输入框____________________] Hint │
│             ↑                              ↑     │
│            4px                           24px   │
│                                                  │
│            ←——————————— 32px ——————————————→    │
│ [标签文字:] [____输入框____________________] Hint │
└─────────────────────────────────────────────────┘
```

| 间距项 | 值 |
|--------|-----|
| 标签文字到冒号（`:`）间距 | `4px` |
| 标签区域固定宽度 | `96px` |
| 输入框右侧到 Hint 文字 | `24px` |
| 相邻表单行的垂直间距 | `32px` |

### 3.2 堆叠标签模式（PhonePad/Phone，≤840px）

```
┌─────────────────────────────┐
│ 标签文字                     │
│ ↕ 8px                       │
│ [____输入框_______________]  │
│ ↕ 12px                      │
│ 标签文字                     │
│ ↕ 8px                       │
│ [____输入框_______________]  │
│ ↕ 12px                      │
│ 标签文字                     │
│ Hint 说明文字                 │
└─────────────────────────────┘
```

| 间距项 | 值 |
|--------|-----|
| 标签文字到输入框的垂直间距 | `8px` |
| 相邻表单行的垂直间距 | `12px` |
| Hint 文字在输入框下方 | `12px` |

## 四、多列表单的栅格规则

### 4.1 各断点每行项数与列宽

| 断点范围 | 栅格总列数 | 每行表单项数 | 每项宽度（栅格 token） | 标签位置 |
|---------|-----------|------------|---------------------|---------|
| >1680px（超大屏/桌面） | 24 列 | **4 项/行** | `--o-r-grid-6` | 左置（水平） |
| 1201–1680px（笔记本） | 24 列 | **4 项/行** | `--o-r-grid-6` | 左置（水平） |
| 841–1200px（平板横屏） | 12 列 | **3 项/行** | `--o-r-grid-4` | 左置（水平） |
| 601–840px（平板竖屏） | — | **1 项/行** | 全宽 | 上置（堆叠） |
| 360–600px（手机） | 4 列 | **1 项/行** | 全宽 | 上置（堆叠） |

> **注**：841–1200px 仍使用水平（左置）标签，与 PC 保持一致，仅每行项数减少到 3 项。

### 4.2 多列布局示意（桌面端，4 项/行）

```
┌──────────────────────────────────────────────────────────────────┐
│ ┌────────────┐  ┌────────────┐  ┌────────────┐  ┌────────────┐  │
│ │ grid-6     │  │ grid-6     │  │ grid-6     │  │ grid-6     │  │
│ │ [字段 1]   │  │ [字段 2]   │  │ [字段 3]   │  │ [字段 4]   │  │
│ └────────────┘  └────────────┘  └────────────┘  └────────────┘  │
│       ↑ 水槽 ↑       ↑ 水槽 ↑       ↑ 水槽 ↑                      │
│                                                                  │
│ ┌────────────┐  ┌────────────┐  ┌────────────┐  ┌────────────┐  │
│ │ grid-6     │  │ grid-6     │  │ grid-6     │  │ grid-6     │  │
│ │ [字段 5]   │  │ [字段 6]   │  │ [字段 7]   │  │ [字段 8]   │  │
│ └────────────┘  └────────────┘  └────────────┘  └────────────┘  │
│          ↑ 行间距 32px ↑                                          │
└──────────────────────────────────────────────────────────────────┘
4×grid-6 + 3×水槽 = 4×424 + 3×32 = 1792px = 24列内容区（1920px画板）✓
```

### 4.3 宽项（跨行元素）的处理

文本域（`OTextarea`）、文件上传（`OUpload`）等需要较大空间的控件，**独占一整行**：

```
┌──────────────────────────────────────────────────────────────────┐
│ ┌────────────┐  ┌────────────┐  ┌────────────┐                  │
│ │ [字段 1]   │  │ [字段 2]   │  │ [字段 3]   │  （空置）         │
│ └────────────┘  └────────────┘  └────────────┘                  │
│ ┌────────────────────────────────────────────────────────────┐   │
│ │ [文本域 / 上传区域                                    全宽] │   │
│ └────────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────┘
```

## 五、表单分组约束

### 5.1 单组项目数量

| 类型 | 数量限制 |
|------|---------|
| 标准多列分组（section）| **2–7 个**表单项 |
| 单列宽项（文本域、上传等）| 不计入列数限制 |

### 5.2 向导式（步骤）表单的分组

- 步骤数：建议 **2–7 步**
- 每步专注 1 个主题，字段数不宜超过 1 行（即 ≤4 个字段）

## 六、特殊展示形式：只读/详情表单

设计文档中包含一种**只读展示型表单**，用于显示已录入的结构化数据（如设备规格），不含输入控件：

```
┌─────────────────────────────────────────────────────────────┐
│ AI     │ 8 TOPS INT8 / 4 TFLOPS FP16                        │
│ 内存   │ LPDDR4X 4GB ECC                                    │
│ CPU    │ 4 core × 1.0 GHz                                   │
│ 解码器 │ H.264/H.265 Decoder 20路1080P@30FPS                 │
│        │ 2路4K@75FPS / JPEG 1080P@512FPS                    │
└─────────────────────────────────────────────────────────────┘
```

此类场景使用 `OForm layout="h"` 配合只读输入框或纯文本展示。
